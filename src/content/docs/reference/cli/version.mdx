Apply all pending changesets to produce a new versioned release: bump versions, snapshot specs, and generate changelogs.

## Usage

```sh
contractual version [--dry-run]
```

## Description

`contractual version` is the release command. It reads every changeset file in `.contractual/changesets/`, flattens their bump types (highest wins per contract), then performs the following steps atomically:

1. **Resolve bump types.** For each contract, the highest bump across all pending changesets wins: `major` > `minor` > `patch`.
2. **Update versions.** Writes new version numbers to `.contractual/versions.json`.
3. **Snapshot specs.** Copies the current state of each spec into `.contractual/snapshots/` tagged with the new version.
4. **Generate changelog.** Appends a new entry to `CHANGELOG.md` at the project root, grouped by contract and bump type.
5. **Delete consumed changesets.** Removes the processed files from `.contractual/changesets/`.

After `version` completes, you commit the resulting changes (`contractual.yaml`, `versions.json`, `CHANGELOG.md`, snapshots, and the removal of changeset files) as your release commit.

Use `--dry-run` to preview what would happen without writing anything to disk. This is useful in CI to validate that pending changesets are self-consistent before merging to the release branch.

## Options

| Flag | Description | Default |
|---|---|---|
| `--dry-run` | Compute and display the version bump plan without writing any files. | Off |

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Versions bumped and all output files written successfully (or `--dry-run` completed). |
| `2` | Configuration error — e.g. `contractual.yaml` not found, malformed changeset file, or no changesets found to consume. |
