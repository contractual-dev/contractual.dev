See at a glance which contracts have pending changesets and what their next version will be.

## Usage

```sh
contractual status [--format <fmt>]
```

## Description

`contractual status` reads `.contractual/versions.json` for current versions and scans `.contractual/changesets/` for pending changeset files. For each registered contract it shows:

- The **current version** from the last `contractual version` run.
- Any **pending changesets** that have not yet been consumed, grouped by contract.
- The **projected next version** based on the highest pending bump type.

Contracts with no pending changesets are listed with a clean status. The command exits with code `1` when any contract has pending changesets, and `0` when there is nothing to release. This makes `status` useful as a CI gate — for example, to remind contributors to run `contractual version` before merging a release branch.

## Options

| Flag | Description | Default |
|---|---|---|
| `--format <fmt>` | Output format. One of `text`, `json`. | `text` |

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | No pending changesets — all contracts are up to date. |
| `1` | One or more contracts have pending changesets waiting to be consumed by `contractual version`. |
